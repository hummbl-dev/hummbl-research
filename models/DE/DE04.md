# DE04 - Dependency Mapping

> Identify and visualize relationships between components.

## Description

Dependency Mapping decomposes systems by charting how elements rely on each other. It reveals bottlenecks, parallel paths, and critical chains, enabling better sequencing and risk management.

## Example

In software, map dependencies: UI depends on API, API on database. This shows that database issues block everything, prioritizing its stability.
